# 3. 通信类 API — 全功能频道

该模块基于 **WebSocket**。

---

通过如下 ws地址 建立连接
```
/ws/chat?token={token}
```

token 通过接口 `/api/user/token` 获取。

所有通信数据均为 **JSON 格式**。

所有消息均包含 `command` 字段用于区分类型。

---

# 3.0 消息对象结构

文档中统一将以下 JSON 结构称为 **消息对象**：

```json
{
  "sendId": 10001,
  "recId": 10002,
  "html": "消息内容",
  "ts": 1773410571322,
  "command": 0
}
```

字段说明：

| 字段      | 类型     | 说明           |
|---------|--------|--------------|
| sendId  | long   | 消息发送者ID      |
| recId   | long   | 消息接收者ID      |
| html    | string | 消息内容（HTML格式） |
| ts      | long   | 消息发送时间       |
| command | int    | 消息类型         |

---

服务器推送下面的数据，客户端可实现 command 0 的处理函数接收实时消息：

```json
{
  "sendId": 10006,
  "recId": 10001,
  "html": "12",
  "ts": 1773034486233,
  "command": 0
}
```

---

# 3.1 获取历史消息

服务器会 **批量发送消息对象**。

每条消息：

```
command = 1
```

最后一条消息：

```
"last": true
```

示例：

```
{"sendId":10005,"recId":10001,"html":"我同意了你的好友申请！","ts":1772686360998,"command":1}
...
{"sendId":10001,"recId":10005,"html":"终于找到安装包了","ts":1772867524143,"command":1,"last":true}
```

---

# 3.2 标为会话为未读

客户端发送：

```json
{
  "command": 3,
  "sendId": 10006
}
```

说明：

* `sendId` 为 **需要标记已读的会话用户ID**

服务器返回：

> 注意 这只会返回给调用的客户端，不会返回给所有的客户端，更不推荐客户端使用这个 API 来更新
> 
> 因为这个需要客户端主动调用，但其实在客户端主动调用之前就知道是谁发了未读消息了，所以根本没必要走这个 API 来获取信息
> 
> 这个 API 的存在是为了标注未读会话的，回复的数据不太需要

```json
{
  "command": 2,
  "needReadMessageFromId": 10006,
  "lastMessage": {
    "html": "12",
    "recId": 10001,
    "sendId": 10006,
    "ts": 1773034486233
  }
}
```

字段说明：

| 字段                    | 说明          |
|-----------------------|-------------|
| needReadMessageFromId | 未读消息来自的会话ID |
| lastMessage           | 最后一条消息      |

---

# 3.3 标记会话为已读

> 与标记为未读一致，专用来将一个会话标记为已读，这样在其他设备上也会同步读取状态

客户端发送：

```json
{
  "command": 3,
  "sendId": 10006
}
```

说明：

* `sendId` 为 **需要标记已读的会话用户ID**

---

# 3.4 获取在线设备数量

客户端请求：

```json
{
  "command": 4
}
```

服务器回复：

```json
{
  "command": 4,
  "count": 2
}
```

字段说明：

| 字段    | 说明         |
|-------|------------|
| count | 当前用户在线设备数量 |

---

# 3.5 Token 占位回传

该命令 **无需客户端主动调用**。

主要用于 **Android 原生环境获取 token**。

客户端发送：

```json
{
  "sendId": 1,
  "recId": 10001,
  "html": "token",
  "ts": 123456789,
  "command": 5
}
```

客户端环境需支持：

```javascript
window.CodeBookApp && typeof window.CodeBookApp.notify === 'function'
```

---

# 3.6 获取所有未读消息

客户端发送：

```json
{
  "command": 6
}
```

服务器返回 **批量消息对象**。

最后一条消息：

```
"last": true
```

如果没有未读消息：

服务器 **不会返回任何数据**。

---

# 3.7 Token 刷新

客户端发送：

```json
{
  "command": 7,
  "token": "旧 token"
}
```

服务器返回：

```json
{
  "command": 7,
  "token": "新 token 或 原 token"
}
```

说明：

服务器会根据策略决定是否刷新 token。

---

# 3.8 服务端强制退出登录

服务器推送：

```json
{
  "command": 8
}
```

客户端收到后应 **执行退出登录逻辑**。

---

# 3.9 服务端主动推送在线状态

客户端无需调用，由服务器主动推送，推送数据为自己好友的在线状态，当自己好友的状态可能发生变化的时候，服务器就会推送。

服务器推送：

```json
{
  "command": 9,
  "userId": 10003,
  "online": true
}
```

其中 `userId` 对应的是可能发生在线状态变化的好友的ID，`online` 对应的是好友的在线状态

客户端应根据 `online` 更新好友在线状态
